# 酷家乐订单后处理

## 核心流程

1. 确认脚本由订单字段、提审页面还是提交审核触发。
2. 明确入口函数签名和订单设置中的绑定关系。
3. 从 `DcsOrder` 获取模型和订单数据。
4. 先过滤目标模型，再进行统计或聚合。
5. 返回字段要求的基础类型。
6. 用订单服务日志验证触发和结果。

## 安全要求

- 案例不得提交客户、订单、电话和地址数据。
- 日志只保留最小脱敏片段。
- 主材、价格和供应链映射使用占位符。
