# 日志与错误排查

## 日志模板

```ts
import { printSystemLog } from "system";

printSystemLog("业务前缀：0 start");
printSystemLog(["业务前缀：1 变量", value].join(" - "));
```

日志显示顺序可能和执行顺序不同，因此必须带序号。

## 未找到定义

依次检查：

1. import 模块是否正确；
2. 当前业务模块是否暴露该类型；
3. 自定义对象是否已发布；
4. 子脚本唯一标识和函数名是否一致；
5. 调用者是否在依赖发布后重新保存发布。

## 唯一标识已存在

系统入口通常只能有一个。查找并编辑已有脚本，不要通过改名绕过。

## 格式错误或自定义代码无效

1. 缩减为最小函数；
2. 删除不支持语法；
3. 检查重复变量名；
4. 检查数组/对象最后一项多余逗号；
5. 在浏览器 Network 查看 `/kb-editor/api/customdef` 返回；
6. 逐段恢复代码定位问题。

## 运行无结果

区分：

- 没触发；
- 已触发但条件未命中；
- 已构建结果但输出链路未采纳；
- 商品或模型无效。

通过入口日志、条件日志、创建日志、返回数量和最终 JSON 逐层判断。
